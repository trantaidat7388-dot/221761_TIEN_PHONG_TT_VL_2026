# Báo Cáo Sửa Lỗi và Cập Nhật Dự Án Word2LaTeX
**Thời gian hoàn thành:** 04/03/2026 20:31

Tài liệu này tổng hợp toàn bộ các vấn đề (lỗi) đã được phát hiện trong quá trình biên dịch file Word mẫu (`-TUAN_Magazine...docx`) sang chuẩn IEEE và Springer, nguyên nhân gốc rễ của từng lỗi, và các giải pháp đã được áp dụng vào mã nguồn.

---

## 1. Lỗi Không Biên Dịch Được PDF (Missing Document Classes)
*   **Nguyên nhân gốc rễ:** Hệ thống thiếu các file định dạng cốt lõi của LaTeX là `IEEEtran.cls` (cho chuẩn IEEE) và `llncs.cls` (cho chuẩn Springer LNCS). Khi gọi lệnh `xelatex`, chương trình báo lỗi `File 'IEEEtran.cls' not found` và dừng biên dịch lập tức.
*   **Thực hiện thay đổi:** 
    *   Tải và bổ sung file `IEEEtran.cls` vào thư mục `backend/custom_templates/`.
    *   Tải và bổ sung file `llncs.cls` vào thư mục `backend/custom_templates/`.
    *   Cập nhật `TEMPLATE_MAP` trong `backend/main.py` để sửa mapping của `springer_lncs` trỏ đúng vào file `samplepaper.tex` hiện có, thay vì file ảo `splnproc1110.tex`.

## 2. Quá Trình Nhúng Ảnh Vào PDF Bị Lỗi (Không Hiện Ảnh)
*   **Nguyên nhân gốc rễ:** Thư mục giải nén ảnh có tên được lấy từ tên file Word gốc (VD: `-TUAN_Magazine_01-12-..._1__20260304`). Tên file này chứa nhiều khoảng trắng (dấu cách). Trình biên dịch LaTeX (`\includegraphics`) không hỗ trợ đọc các tệp tin trong thư mục có khoảng trắng trắng, dẫn đến file PDF được tạo ra nhưng hoàn toàn trống ảnh.
*   **Thực hiện thay đổi:**
    *   Sửa logic trong `backend/main.py` ở API `/api/chuyen-doi`: Ép buộc đổi tên thư mục giải nén ảnh thành một tên tĩnh là `"images"` bên trong Job Folder.
    *   Nhờ đó lệnh trong file `.tex` luôn là `\includegraphics{images/...}`, đảm bảo tuyệt đối không có khoảng trắng trong đường dẫn.

## 3. Lỗi Frontend Hiển Thị "0 Trang" Dù PDF Có Trang
*   **Nguyên nhân gốc rễ:** Hàm phân tích log của LaTeX (`re.search(r'Output written on .*\((.*?) pages', log)`) trong đoạn code đếm số trang trả về rỗng vì dòng cớ trong file `output.log` bị ngắt (xuống hàng) giữa chừng khi đường dẫn file quá dài. Ngoài ra, file log có thể bị xóa sớm bởi hàm dọn dẹp trước khi đọc.
*   **Thực hiện thay đổi:**
    *   Thêm tuỳ chọn `re.DOTALL` vào Regex để cho phép tìm kiếm qua các dấu xuống dòng.
    *   Bổ sung phương pháp đệm (fallback): Đọc trực tiếp định dạng nhị phân (binary) của tệp tin `.pdf` để đếm số lượng thẻ `/Type /Page` nếu Regex file log bị thất bại.

## 4. Các Sơ Đồ Đơn Giản Bị Lọc Sai Thành "Ảnh Trang Trí"
*   **Nguyên nhân gốc rễ:** Thuật toán phát hiện ảnh trang trí trong thư viện tính kích thước ảnh có ngưỡng chưa chính xác, loại bỏ nhầm các sơ đồ quan trọng có kích thước/dung lượng nhỏ nhưng lại là sơ đồ khối mạch lạc của hệ thống.
*   **Thực hiện thay đổi:**
    *   Đơn giản hoá hoàn toàn hàm `la_anh_trang_tri()` trong tệp `src/xu_ly_anh.py`. Chỉnh hàm luôn trả về `False` để tạm thời vô hiệu hoá bộ lọc này, qua đó giữ lại 100% hình ảnh (dù to hay nhỏ), đảm bảo toàn vẹn sơ đồ khoa học trong báo cáo thuật toán.

## 5. Cột Tài Liệu Tham Khảo (References) Bị Tràn Lề Phải (Overflow)
*   **Nguyên nhân gốc rễ:** Các đường link website (URL) dài (ví dụ các link GitHub hoặc bài báo) trong mục References không thể tự động bẻ dòng trong LaTeX nếu nó vượt quá độ rộng của 1 cột, vì nó được chèn dưới dạng Plain Text.
*   **Thực hiện thay đổi:**
    *   Bổ sung sử dụng các thư viện `\usepackage{url}` và `\usepackage{hyperref}` vào giao diện cả 2 template IEEE (`IEEE-conference-template-062824.tex`) và Springer (`samplepaper.tex`).
    *   Viết thêm regex xử lý toàn cục trong `src/utils.py` (hàm `loc_ky_tu()`) để bắt mọi chuỗi bắt đầu bằng `http://` hoặc `https://` và tự động bọc chuỗi đó vào lệnh `\url{...}`. Qua đó cho phép các đường ranh giới tại dấu `/` và dấu `-` tự động bẻ xuống dòng sao cho khít lề.

## 6. Dư Trữ Thông Tin Tác Giả (Dummy Authors / Dummy Institutes)
*   **Nguyên nhân gốc rễ:** Trong template gốc (của hiệp hội IEEE và Springer), có chèn sẵn dữ liệu mẫu về các tác giả và các trường Đại Học. Đoạn code tiêm dữ liệu trước đây (`src/chuyen_doi.py` - hàm `_thay_the_author`) bằng Regex chỉ xử lý thay block `\author{}` thứ nhất nhưng không xóa lây sang các thẻ rác ở đuôi (như `\institute` và `\and`), dẫn đến việc file LaTeX có cả tác giả mới và thông tin rác của tác giả mẫu chồng lên ở cuối.
*   **Thực hiện thay đổi:**
    *   Mở file template `IEEE-conference-template-062824.tex` xoá sạch 5 dòng tác giả nháp, giữ lại mẫu chuẩn dùng làm khung tiêm dữ liệu.
    *   Cập nhật `_thay_the_author` bằng hàm xoá triệt để (while loop) tất cả các thẻ `\institute`, `\and`, `\affil` trước khi tự động chèn dữ liệu tác giả vừa lấy từ Word. Đảm bảo file Word cung cấp thế nào, PDF ra đúng ngần đó tác giả.

## 7. Tự Động Vá Lỗi Tràn Link Cho Template Lạ
*   **Vấn đề:** User lấy template lạ ở ngoài vào, template đó thường không bao gồm sẵn lệnh `\usepackage{url}` hay `hyperref`, kết quả là sinh PDF bị tràn chữ.
*   **Thực hiện thay đổi:** Backend (file `chuyen_doi.py`) giờ đây sẽ tự động quét Preamble của mọi template được nạp vào. Nếu phát hiện thiếu `url` hoặc `hyperref`, Python sẽ **tự động chèn chìm** các gói này vào trước lệnh `\begin{document}`, giúp định dạng link luôn hoàn hảo và an toàn.

## 8. Hỗ Trợ Cấu Trúc `\abstract{...}` Đời Cũ
*   **Vấn đề:** Chệch form Tóm tắt. Một số template không xài `\begin{abstract}` mà xài `\abstract{...}` cục bộ.
*   **Thực hiện thay đổi:** Thuật toán AI của Python đã được cập nhật thêm Regex pattern cho phép nhận diện và thay thế chính xác nội dung kể cả khi template xài chuẩn `\abstract{...}`.

## 9. Báo Cáo Thiếu File Class (.cls) Trực Tiếp Ra Màn Hình
*   **Vấn đề:** Trước đây nếu thiếu `.cls`, LaTeX ngưng chạy, trả về file ZIP nhưng frontend rơi vào im lặng, người dùng không hiểu tại sao file bị lỗi trống rỗng.
*   **Thực hiện thay đổi:** Hàm biên dịch LaTeX đã được viết lại. Giờ đây nó sẽ quét Output Log Của XeLaTeX. Nếu bắt được dòng text `! LaTeX Error: File XYZ.cls not found`, hệ thống ngay lập tức sẽ ném một HTTP Code 400 ra mặt tiền của App kèm lời nhắn cụ thể *"Thiếu thư viện cấu trúc LaTeX: XYZ.cls. Vui lòng tải file này vào thư mục"*. Người dùng sẽ biết chính xác họ đang làm sai ở đâu mà tự sửa chữa.
